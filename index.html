Examples

Expose Makefile commands safely

Makefile targets become tools

OpenCuff discovers targets and exposes them as namespaced tools like makefile.make_test. 

.PHONY: build test lint

## Build the project
build:
  cargo build --release

## Run tests
test:
  cargo test

Short configuration

Choose which targets are exposed and keep discovery fast and safe. 

plugins:
  makefile:
    type: in_source
    module: opencuff.plugins.builtin.makefile
    config:
      makefile_path: ./Makefile
      targets: "build,test,lint"
      extractor: simple

pnpm scripts become tools

Package.json scripts are exposed as tools like packagejson.pnpm_test. 

{
  "scripts": {
    "build": "tsc -p .",
    "test": "vitest run",
    "lint": "eslint ."
  }
}

Short configuration

Auto-detect pnpm or set it explicitly and filter scripts. 

plugins:
  packagejson:
    type: in_source
    module: opencuff.plugins.builtin.packagejson
    config:
      package_json_path: ./package.json
      package_manager: pnpm
      scripts: "build,test,lint"

Shell scripts become tools

Match trusted scripts and expose them as tools like scripts.deploy. 

scripts/
  deploy.sh
  lint.sh
  release.sh

Short configuration

Use glob patterns to expose only approved scripts. 

plugins:
  scripts:
    type: in_source
    module: opencuff.plugins.builtin.scripts
    config:
      patterns:
        - "scripts/*.sh"
      exclude:
        - "scripts/internal_*.sh"

For untrusted Makefiles, set trust_makefile: false to avoid parsing that executes $(shell ...).

Plugin system

Extend OpenCuff without slowing the core

In-source plugins

Fast Python plugins loaded directly inside OpenCuff.

Process plugins

Language-agnostic tools over stdin/stdout JSON.

HTTP plugins

Remote services called over HTTP for scale or isolation.

Tools are namespaced by plugin name (for example, makefile.make_test), and config changes hot-reload without restarting the server.

Cuff CLI

Fast setup with the Cuff command line

Initialize configs

cuff init scans your repo and generates a settings.yml with Makefile, package.json, and script plugins pre-filled.

Run OpenCuff

Start the MCP server locally with stdio, or switch to SSE for remote clients. 

cuff init
cuff run
cuff run -t sse -p 8080
